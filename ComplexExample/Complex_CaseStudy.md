# Complex Case Study: Multi-Visit Persistence in a Nonlinear Platoon

This document details an extended, non-vacuous computational example of Path-Complete Closure Certificates (PC-CC) applied to a nonlinear discrete-time switched system. 

Unlike the simple illustrative example provided in the main manuscript (where trajectories leave the finite-visit set after a single step), this scenario modifies the initial and finite-visit regions to enforce a **complex, multi-step descent**. Trajectories can visit the finite-visit set up to **four consecutive times** before escaping. 

This scenario serves to explicitly demonstrate the non-vacuous application of the PC-CC well-foundedness condition (PC-CC3) and highlights the structural scalability of the PC-CC framework when addressing complex dynamical switching.

## 1. System Dynamics

We consider a two-vehicle platoon system where the state vector $x = [x_1, x_2]^\top$ consists of the follower ($x_1$) and leader ($x_2$) velocities. The system switches arbitrarily between two operational modes: Mode 1 (normal communication) and Mode 2 (communication breakdown). 

The discrete-time polynomial dynamics are given by:

$$
f_1(x) = 
\begin{bmatrix}
0.01 x_2 + 0.9 x_1 - 0.02 x_1^2 \\
2.0 + 0.8 x_2 - 0.04 x_2^2
\end{bmatrix}
$$

$$
f_2(x) = 
\begin{bmatrix}
0.9 x_1 - 0.02 x_1^2 \\
2.0 + 0.8 x_2 - 0.04 x_2^2
\end{bmatrix}
$$

## 2. State Space and Target Sets

The system is bounded within a compact state space $\mathcal{X}$, where velocities are constrained between $0$ and $6$, and the leader is always moving at least as fast as the follower ($x_2 \ge x_1$):

$$
\mathcal{X} = \{x \in \mathbb{R}^2 : 0 \le x_1 \le 6, \; 0 \le x_2 \le 6, \; x_2 - x_1 \ge 0\}
$$

Let $g(x) = x_2 - x_1$ denote the inter-vehicle gap. The initial state set $\mathcal{X}_0$ and the finite-visit set $\mathcal{X}_{\mathrm{vf}}$ are defined by entirely different gap bounds than the simple example:

$$
\mathcal{X}_0 = \{x \in \mathcal{X} : 2.5 \le g(x) \le 3.0\}
$$

$$
\mathcal{X}_{\mathrm{vf}} = \{x \in \mathcal{X} : 1.0 \le g(x) \le 2.5\}
$$

By enforcing these specific bounds, the system's trajectories evolve inside the intermediate gap boundary for up to **4 consecutive time steps** before the nonlinear dynamics naturally spread the vehicles apart and permanently drive them out of $\mathcal{X}_{\mathrm{vf}}$.

## 3. PC-CC Synthesis and Results

To formally verify persistence with respect to $\mathcal{X}_{\mathrm{vf}}$ under arbitrary switching, we employed the PC-CC Sum-of-Squares (SOS) framework. We provide two configurations to capture the arbitrary switching between the two modes:
1. **One-Node Graph (Common Certificate):** Evaluates a single, global certificate function across all mode transitions.
2. **Two-Node Graph (Path-Complete Graph):** Distributes the certificates across two distinct graph nodes, enabling state-dependent verification for specific transition sequences.

Because the dynamics execute a complex, four-step trajectory sequence prior to escaping the set, the geometric structure requires significant expressive power to bound. For both graph configurations, we successfully synthesized a **Degree 6** Path-Complete Closure Certificate ($C_{p,q}(x,y) \in \mathrm{P}[x,y]_6$) using standard, fixed scalar S-procedure multipliers ($\tau = 1$). 

The successful extraction of the degree-6 certificates formally proves that:
1. The trajectories visit $\mathcal{X}_{\mathrm{vf}}$ only finitely many times.
2. The strict decrease condition (PC-CC3) was utilized **non-vacuously**, operating actively over the 4-step sequence to guarantee eventual escape.

*(The full, raw degree-6 polynomial certificate coefficients are generated by the provided scripts in this repository: `pccc_onenode.jl` for the one-node graph and `pccc_twonode.jl` for the two-node graph).*
